# Task Cards: Sprint Theia Migration

任务卡片按阶段分组（P0 → P3）：

- P0：PoC 验证（先定生死：输入/快捷键、native、存储语义）
- P1：框架搭建（脚手架 + 裁剪 + Editor Widget 基础 + 布局/品牌）
- P2：核心迁移（RPC/IPC、SQLite、RAG、Embedding）
- P3：创作面板（AI/版本历史/知识图谱面板迁移）

每张 task card 必须包含：

- Context（任务背景）
- Requirements（必须完成的具体条目）
- Acceptance Criteria（完成标准，必须可验收）
- Dependencies（依赖关系）
- Estimated Effort（工作量估算）
